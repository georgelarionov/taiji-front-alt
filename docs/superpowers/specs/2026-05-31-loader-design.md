# Лоудер сайта — дизайн

Дата: 2026-05-31
Статус: одобрено к реализации

## Обзор

Декоративный брендовый сплэш-экран при заходе на сайт. Закрывает вьюпорт на
весь экран (`100vw × 100svh`), плавно проявляет элементы, одновременно
«рисует» каллиграфический круг по его траектории, держит паузу и уезжает вверх.
Фиксированная длительность (~4с, не зависит от реальной загрузки страницы).

Источник макета: фрейм `loader` в `design/design.pen` (артборд 1440×750).

## Границы (scope)

- **НЕ трогаем `Hero.astro`** — ни `data-rise` интро, ни видео. Состыковку старта
  hero с уходом лоудера делает пользователь отдельно.
- Контракт связи: на завершении лоудер диспатчит `CustomEvent('taiji:loader-done')`
  на `document`. Это единственная точка интеграции для будущего кода hero.
- Mobile-вёрстка лоудера — best-effort (фиксированные размеры, симметричное
  центрирование), отдельная мобильная адаптация вне scope.

## Разметка и раскладка (`Loader.astro`)

Контейнер `[data-loader]`: `position:fixed inset-0`, `z` выше всего контента,
фон `bg-surface-sunken` (#f2f1ef). Элементы спозиционированы симметрично от
центра, чтобы адаптироваться к любой высоте вьюпорта (не привязываемся к 750px):

- **Верх-центр** — иероглифика `传统太极拳学会` (`loader-top.png`, 85×17),
  `opacity: 0.5`, прижата к верху (`top: 24px`), центр по X. Плоский растр → обычный `<img>`.
- **Центр** — группа «круг + логотип»:
  - инлайн-`<svg viewBox="0 0 304 304">` с залитым путём круга (`fill #023C97`) —
    это тонкая (~2px) рисованная почти-окружность с крошечным разрывом сверху и
    засечкой справа (не толстый мазок). Путь обёрнут в `<g mask="url(#loader-ring-mask)">`;
    маска — `<circle cx=152 cy=152 r≈150 pathLength=100>`, `fill:none`, `stroke:#fff`,
    `stroke-width` с запасом на тонкую линию + рисованную «гуляющую» геометрию
    (ориентир 12, подогнать), `stroke-dasharray=100`, `stroke-dashoffset=100` (старт скрыт);
    `transform="rotate(-90 152 152)"` чтобы отрисовка стартовала сверху;
  - логотип (`loader-icon.png`, 90×116) — обычный `<img>`, центр по центру круга.
- **Лево-центр / Право-центр** — `2017` (`left:24px`) и `2026` (`right:24px`),
  по вертикали — центр. Inter, 12px, weight 600, `#797978`, letterSpacing -0.12,
  lineHeight 1.2.
- **Низ-центр** — `ОБЩЕСТВО ИЗУЧЕНИЯ\nТРАДИЦИОННОГО ТАЙЦЗИЦЮАНЬ`, прижат к низу
  (`bottom: 24px`), центр по X. Manrope (`font-display`), 18px, weight 700,
  `#797978`, letterSpacing -0.18, lineHeight 1.2, `text-center`.

Токены: фон → `bg-surface-sunken`, круг → акцент `#023C97` (= `--color-accent`).
Серый текста `#797978` — одноразовое значение, через arbitrary `text-[#797978]`
(кандидат на токен `--color-muted`, но не в этой задаче).

Все анимируемые элементы помечаются `data-loader-fade` (fade-in), круг-маска —
отдельной ссылкой для GSAP.

## Отрисовка круга (механизм)

Линия круга — залитый путь (fill, не stroke), поэтому напрямую `dashoffset` к ней
неприменим. Рисуем через SVG-маску:

1. Видимый залитый путь круга лежит под `<g mask="url(#loader-ring-mask)">`.
2. Маска = `<circle>` (обводка) по средней линии кольца, `pathLength=100`,
   `dasharray=100`. Анимируем `stroke-dashoffset` 100 → 0 → линия проявляется вдоль дуги.
3. `stroke-width` маски ≥ толщины линии + запас на рисованную «гуляющую» геометрию
   (ориентир 12; узкая линия ≈2px → 12px маски с запасом перекрывает её везде).
4. `transform="rotate(-90 152 152)"` сдвигает старт дуги к верху (разрыв энсо).
   Направление обхода под «сверху→вниз» — подгоняется визуально (знак dashoffset /
   доп. `scale(-1,1)`).

Только free-плагины GSAP (анимация атрибута), без платного DrawSVGPlugin.
Кросс-браузерно — приоритет проекта.

## Поведение и lifecycle (FOUC-safe, без флэша)

Приём аналогичен `html.js` в проекте. Инлайн `is:inline` скрипт в `<head>`
(исполняется до отрисовки `<body>`) добавляет класс `html.loading` **только
если** одновременно:
- в `sessionStorage` нет флага `taiji:loaded`, **и**
- не активен `prefers-reduced-motion: reduce`.

CSS:
- лоудер виден (и перекрывает контент) только при `html.loading`; иначе `display:none`;
- `html.loading { overflow: hidden }` — блокирует нативный скролл (и Lenis) на время лоудера;
- круг (маска) в начальном состоянии скрыт.

Сценарии:
- **Первая загрузка сессии:** класс ставится до пейнта → лоудер сразу перекрывает,
  контент не мигает; JS отыгрывает таймлайн.
- **SPA-переход (ClientRouter):** инлайн `<head>`-скрипты не перезапускаются →
  класса нет → лоудер `display:none`, флэша нет.
- **Reload в той же сессии / reduced-motion:** класс не ставится → лоудера нет,
  контент сразу виден.
- **Без JS:** класс не ставится (его ставит JS-инлайн) → лоудера нет, контент виден.

Рантайм `src/lib/loader.ts` по контракту проекта:
- слушатели регистрируются на верхнем уровне модуля;
- `astro:page-load` → `run()`: если `html.loading` отсутствует — выходим
  (SPA/reload/reduced-motion/no-JS уже отфильтрованы классом); иначе строим таймлайн;
- `astro:before-swap` → `cleanup()`: убить таймлайн.

По завершении таймлайна `run()`:
1. `sessionStorage.setItem('taiji:loaded', '1')`;
2. снять `html.loading` (разблок скролла) и спрятать лоудер;
3. `document.dispatchEvent(new CustomEvent('taiji:loader-done'))`;
4. убить таймлайн.

## Тайминг

Константы — в начале `loader.ts`, легко менять. Ориентир:

- `t0`: fade-in всех `data-loader-fade` (~0.8с, лёгкий y-rise + opacity, малый stagger);
- `t0` (одновременно): отрисовка круга (~2.4с, ease из референса);
- hold до ~3.2с;
- уход вверх: `translateY(-100%)` всего контейнера (~0.9с), ease `CustomEase("loader", "0.65, 0.01, 0.05, 0.99")` (free CustomEase);
- итого ≈ 4.1с (подгоняемо).

`prefers-reduced-motion` — таймлайн не строится (лоудер уже скрыт классом-гейтом),
для надёжности JS дополнительно гейтит.

## Зависимости

Используем уже установленный `gsap` (стоит в `apps/web`). Доп. плагин —
`gsap/CustomEase` (входит в ядро GSAP 3.12+, бесплатный, отдельный пакет не нужен).
При реализации проверить, что импорт `gsap/CustomEase` резолвится в сборке.

## Верификация

- `pnpm check` (TS/Astro диагностика) — обязательно.
- `pnpm build` — обязательно (статическая сборка проходит).
- Визуальную проверку в браузере (Playwright) — только если пользователь попросит явно.
- Ручные пункты (для себя/пользователя): первая загрузка показывает лоудер без
  флэша; SPA-переход и повторная загрузка в сессии лоудер не показывают; круг
  рисуется по дуге; уход вверх; скролл заблокирован на время лоудера; событие
  `taiji:loader-done` диспатчится; `scrollWidth ≤ innerWidth` (нет гориз. скролла).
